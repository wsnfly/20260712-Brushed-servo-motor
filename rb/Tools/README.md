# MODBUS RTU 电机控制器调试工具

本项目是一款基于 Python Tkinter 编写的 **MODBUS RTU 电机控制器调试与配置工具**。它专为支持 MODBUS RTU 协议的电机控制器设计，提供了串口连接、工作模式切换、运动控制、PID参数在线整定、智能复位、多功能引脚配置、实时状态监控以及原始通信报文解析等完整的功能集。

---

## 🚀 功能特性

1. **多重设备与通信管理**
   * **串口自适应扫描**：自动获取系统可用串口，支持常见波特率（9600 至 115200）。
   * **在线设备地址配置**：可读取并修改电机的 MODBUS 从机地址（范围 1~247）。
   * **在线波特率切换**：支持在线修改 STM32 电机控制器的串口波特率，成功后上位机自动同步切换，无需重新连接。

2. **丰富的工作模式支持**
   * **位置模式 (Mode 0)**：精确控制电机转动至指定的 64 位有符号目标位置。
   * **速度模式 (Mode 1)**：控制电机以指定的 32 位有符号速度持续运行。
   * **位置速度模式 (Mode 2)**：基于目标位置和最大运行速度限制进行复合运动控制。
   * **开环模式 (Mode 3)**：直接设定 PWM 输出进行开环驱动。
   * **外部目标位置模式 (Mode 4)**：由外部脉冲或硬件接口给出目标位置。

3. **双环 PID 参数在线整定**
   * 提供**位置环**（Kp, Ki, Kd）和**速度环**（Kp, Ki, Kd）参数的在线读取与写入。
   * 界面参数采用浮点数显示，下发时**自动乘以 100** 转换为 16 位补码（精度 0.01），支持负值参数。

4. **高级运动与保护参数**
   * **死区设置**、**最大输出**（PWM幅值）、**最大运行速度限制**。
   * **方向控制**：独立配置电机运行方向和编码器反馈方向，便于适配不同的机械结构。
   * **启动控制**：支持“直接启动”或“标志位触发（写 0x0010 启动寄存器）”两种启动方式。

5. **智能复位 (Homing) 控制**
   * 支持 3 种复位模式：**关闭**、**堵转复位** 和 **精确复位**。
   * 可配置复位方向（正/负）、复位电流限制（PWM）、复位运行速度、精确检测速度与检测次数、复位最大限制距离、复位回退偏置。
   * 支持**开机自动复位**及手动**一键触发复位**。

6. **数字引脚 (PB4/PB5) 多功能映射**
   * 可自由定义 PB4/PB5 两个输入引脚的功能：**脉冲、方向、复位开关、限位开关、目标位置**。
   * 支持配置有效电平（高有效 / 低有效）以及限位方向（停止正方向 / 停止负方向）。
   * 支持“电平目标位置”预设（64 位），当引脚功能设为“目标位置”且被触发时，电机自动运动到预设的绝对位置。

7. **实时状态监控与双窗口调试**
   * 100ms 高频刷新：实时显示当前位置、目标位置、当前速度、当前模式、启动标志、PWM 输出（带正负极性颜色区分：正为红、负为蓝、零为灰）及复位状态。
   * **通信日志窗口**：记录操作详情与异常报警。
   * **原始数据包窗口 (HEX Stream)**：以高亮颜色区分显示发送（TX 绿色）和接收（RX 蓝色）的原始 Modbus HEX 数据帧，极其便于协议层面的分析与调试。

8. **一键恢复默认值**
   * 可下发指令使电机控制器恢复至出厂默认配置（对齐 `main.c` 固件默认值），并将参数持久化保存到控制器的 Flash 中。

## 🛠️ 安装依赖

程序使用 Python 3 编写，仅依赖 `pyserial` 和内置的 `tkinter`。

```bash
pip install -r requirements.txt
```

> **提示**：如果系统未安装 Python 或 `pyserial`，请先进行安装。

## 🏃 运行程序

双击运行或在控制台中执行以下命令：

```bash
python modbus_debugger.py
```

## 📘 寄存器地址映射表

工具内置了严密的 MODBUS 寄存器地址映射，所有寄存器均为 **Holding Registers (保持寄存器，功能码 03/06/16)**。

### 1. 配置参数区 (可读写)

| 寄存器地址 | 寄存器名称 | 数据类型 | 描述/取值范围 |
| :--- | :--- | :--- | :--- |
| **0x0000** | 模式控制字 | 16-bit | 设定控制模式（0=位置, 1=速度, 2=位置速度, 3=开环, 4=外部目标） |
| **0x0001** | 启动方式设置 | 16-bit | 0 = 直接启动， 1 = 标志位启动 |
| **0x0002 - 0x0005** | 目标位置 | 64-bit 有符号 | 分为 H3, H2, L2, L1 四个 16 位寄存器 |
| **0x0006 - 0x0007** | 目标速度 | 32-bit 有符号 | 分为 H, L 两个 16 位寄存器 |
| **0x0008 - 0x000A** | 位置环 PID | 16-bit * 3 | 依次为 Kp, Ki, Kd （数值 = 实际值 × 100） |
| **0x000B - 0x000D** | 速度环 PID | 16-bit * 3 | 依次为 Kp, Ki, Kd （数值 = 实际值 × 100） |
| **0x000E** | 死区限制 | 16-bit | 允许的定位死区脉冲数 |
| **0x000F** | 最大输出幅值 | 16-bit | 限制最大 PWM 输出幅值 (通常为 0~1000) |
| **0x0010** | 启动触发 | 16-bit | 在“标志位启动”方式下，写 1 触发电机运行 |
| **0x0011** | 电机方向 | 16-bit | 0 = 正转， 1 = 反转 |
| **0x0012** | 编码器方向 | 16-bit | 0 = 正常反馈， 1 = 反转反馈 |
| **0x0013** | 保留寄存器 | 16-bit | 启动标志备份/保留 |
| **0x0014** | MODBUS 从机地址 | 16-bit | 设定控制器的从机地址 (1~247) |
| **0x0015** | MODBUS 波特率 | 16-bit | 设定控制器波特率（实际波特率 / 100，如 1152 表示 115200bps） |
| **0x0016** | 设当前位置为原点 | 16-bit | 写 1 触发：编码器反馈清零，目标位置同步设为 0 |
| **0x0017** | 复位模式 | 16-bit | 0 = 关闭， 1 = 堵转复位， 2 = 精确复位 |
| **0x0018** | 复位方向 | 16-bit | 0 = 负方向， 1 = 正方向 |
| **0x0019** | 复位电流限制 | 16-bit | 堵转复位时的 PWM 电流限制值 |
| **0x001A - 0x001B** | 复位速度 | 32-bit 有符号 | 寻原点过程的快速运行速度 |
| **0x001C - 0x001D** | 复位最大限制距离 | 32-bit | 复位过程中允许运行的最大安全脉冲距离 |
| **0x001E - 0x001F** | 复位回退偏置 | 32-bit 有符号 | 复位完成后回退的偏置脉冲距离 |
| **0x0020** | 触发一次复位 | 16-bit | 写 1 立即触发一次复位对齐流程 |
| **0x0021** | 开机自动复位 | 16-bit | 0 = 关闭， 1 = 开启 |
| **0x0022** | PB4 引脚功能定义 | 16-bit | 0=脉冲, 1=方向, 2=复位开关, 3=限位开关, 4=目标位置 |
| **0x0023** | PB4 引脚有效极性 | 16-bit | 0 = 高电平有效， 1 = 低电平有效 |
| **0x0024** | PB4 引脚限位方向 | 16-bit | 0 = 停止正方向运动， 1 = 停止负方向运动 |
| **0x0025** | PB5 引脚功能定义 | 16-bit | 0=脉冲, 1=方向, 2=复位开关, 3=限位开关, 4=目标位置 |
| **0x0026** | PB5 引脚有效极性 | 16-bit | 0 = 高电平有效， 1 = 低电平有效 |
| **0x0027** | PB5 引脚限位方向 | 16-bit | 0 = 停止正方向运动， 1 = 停止负方向运动 |
| **0x0028 - 0x0029** | 最大运行速度限制 | 32-bit | 限制最大允许运行速度 (0 = 无限制) |
| **0x002A - 0x002D** | PB4 电平目标位置 | 64-bit 有符号 | 引脚被触发时自动运动到的绝对目标位置 |
| **0x002E - 0x0031** | PB5 电平目标位置 | 64-bit 有符号 | 引脚被触发时自动运动到的绝对目标位置 |
| **0x0032 - 0x0033** | 精确复位检测速度 | 32-bit | 精确复位模式下的二次慢速触碰检测速度 |
| **0x0034** | 精确复位检测次数 | 16-bit | 精确复位触碰检测及取平均值的循环次数 |

## 💡 调试指南与注意事项

1. **多线程安全机制**：
   * 监控和手动配置操作共享同一条物理串口链路，代码内部引入了 `threading.Lock` 串口互斥锁，确保在进行高频监控时，手动读取或写入配置指令能安全插队，避免数据帧冲突、错位。
2. **MODBUS 接收容错**：
   * 每次请求发送前都会主动复位（清空）输入缓冲区（`serial.reset_input_buffer()`），确保接收到的响应帧对齐，杜绝因帧不完整或干扰引起的持续性解析错位。
3. **STM32 波特率在线修改流程**：
   * 写入新波特率后，控制器会**先用旧波特率返回写入成功响应**，随后其硬件串口切换至新波特率。上位机收到响应后，自动同步修改本地的 `serial.baudrate` 为新波特率。整个过程无缝衔接。
4. **保存参数到 Flash**：
   * 在电机控制器一侧，大部分配置写入（如波特率、从机地址、PID、恢复默认等）都会触发内部 Flash 存储。请勿进行频繁无意义的高频写入，以免耗尽 STM32 内部 Flash 擦写寿命。
5. **电机原点设置**：
   * 设置“设当前位置为原点”操作会向控制器写入 `1`。由于控制器将当前编码器计数清零，为防止电机在位置模式下因编码器值突变而剧烈抖动，界面上的“目标位置值”会被同步清零，确保运动安全。
