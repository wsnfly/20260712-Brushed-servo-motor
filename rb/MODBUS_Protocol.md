## 2. 帧格式

MODBUS RTU 帧由「从机地址 + 功能码 + 数据 + CRC16」组成，帧与帧之间依靠 3.5 字符的总线空闲时间区分。

```
+----------+--------+------------+--------+
| 从机地址 | 功能码 | 数据区     | CRC16  |
| 1 byte   | 1 byte | N bytes    | 2 byte |
+----------+--------+------------+--------+
```

### 2.1 帧间超时

- TIM1 每 100µs 中断一次（`MODBUS_OnTimeoutTick`）。
- 每收到一个字节，`timeout_ticks` 清零。
- 当 `timeout_ticks >= MB_FRAME_TIMEOUT_TICKS (=4)` 时判定一帧接收完成。
- 115200bps 8N1 下：1 字符 = 10 bit / 115200 ≈ 86.8µs，3.5 字符 ≈ 303.8µs ≈ 4 个 100µs 计数。

### 2.2 CRC16

- 多项式 0xA001，初始值 0xFFFF，查表法实现。
- 发送时 CRC 低字节在前、高字节在后（小端序）。
- 接收校验失败时丢弃帧并累加 `error_count`。

## 3. 支持的功能码

| 功能码 | 名称 | 说明 |
|---|---|---|
| 0x03 | Read Holding Registers | 读保持寄存器，最多 125 个 |
| 0x06 | Write Single Register | 写单个寄存器（写后立即生效） |
| 0x10 | Write Multiple Registers | 写多个寄存器，最多 125 个 |
| 其他 | — | 返回异常码 0x01（非法功能码） |

### 3.1 功能码 0x03 — 读保持寄存器

**请求帧：**

| 字节 | 内容 |
|---|---|
| 0 | 从机地址 |
| 1 | 0x03 |
| 2-3 | 起始地址（大端） |
| 4-5 | 寄存器数量（大端） |
| 6-7 | CRC16 |

**正常响应：**

| 字节 | 内容 |
|---|---|
| 0 | 从机地址 |
| 1 | 0x03 |
| 2 | 字节数 = 寄存器数量 × 2 |
| 3..N | 寄存器数据（每个 2 字节，大端） |
| N+1-N+2 | CRC16 |

### 3.2 功能码 0x06 — 写单个寄存器

**请求帧：**

| 字节 | 内容 |
|---|---|
| 0 | 从机地址 |
| 1 | 0x06 |
| 2-3 | 寄存器地址（大端） |
| 4-5 | 写入值（大端） |
| 6-7 | CRC16 |

**正常响应：** 原样回显请求帧（共 8 字节）。

### 3.3 功能码 0x10 — 写多个寄存器

**请求帧：**

| 字节 | 内容 |
|---|---|
| 0 | 从机地址 |
| 1 | 0x10 |
| 2-3 | 起始地址（大端） |
| 4-5 | 寄存器数量（大端） |
| 6 | 字节数 = 数量 × 2 |
| 7..N | 寄存器数据（每个 2 字节，大端） |
| N+1-N+2 | CRC16 |

**正常响应：**

| 字节 | 内容 |
|---|---|
| 0 | 从机地址 |
| 1 | 0x10 |
| 2-3 | 起始地址 |
| 4-5 | 寄存器数量 |
| 6-7 | CRC16 |

多寄存器写入在整帧写入完成后，若写入了目标位置/速度寄存器，统一调用一次 `MODBUS_ApplyTarget`。这样可避免 64 位目标位置在分 4 次写入过程中被中途应用。

### 3.4 异常响应

| 字节 | 内容 |
|---|---|
| 0 | 从机地址 |
| 1 | 功能码 \| 0x80 |
| 2 | 异常码 |
| 3-4 | CRC16 |

| 异常码 | 名称 | 触发条件 |
|---|---|---|
| 0x01 | Illegal Function | 不支持的功能码 |
| 0x02 | Illegal Data Address | 地址超出允许范围 |
| 0x03 | Illegal Data Value | （预留） |
| 0x04 | Slave Device Failure | （预留） |

## 4. 工作模式

电机控制器支持 5 种工作模式，由 `MotorMode_t` 枚举定义：

| 模式值 | 名称 | 说明 |
|---|---|---|
| 0 | POSITION_MODE（位置模式） | 控制电机转到并保持在目标位置，走位置环 PID |
| 1 | SPEED_MODE（速度模式） | 控制电机以目标速度持续运转，走速度环 PID |
| 2 | VELOCITY_POSITION_MODE（位置速度模式） | 目标位置按目标速度浮点累加，走位置环 PID |
| 3 | OPENLOOP_MODE（开环模式） | 目标速度值直接映射到 PWM 输出 |
| 4 | EXTERNAL_TARGET_MODE（外部目标位置模式） | PB4/PB5 电平选择目标位置，走位置环 PID |

### 4.1 位置模式（POSITION_MODE）

- 通过 REG_TARGET_POS（0x0002~0x0005）设置绝对目标位置。
- 误差 = target_position − current_position。
- 走位置环 PID（pos_Kp/Ki/Kd）。
- 启用死区：`|error| ≤ dead_zone` 时停止输出，防止抖动。
- 误差过零时清零积分，防止超调后积分反向抵消。
- D 项 = −Kd × velocity（用当前速度做微分阻尼）。

### 4.2 速度模式（SPEED_MODE）

- 通过 REG_TARGET_SPEED（0x0006~0x0007）设置目标速度（脉冲/秒）。
- 误差 = target_speed − velocity。
- 走速度环 PID（spd_Kp/Ki/Kd）。
- 速度死区：`|error| < 20` 时误差置 0，防止低速抖动。
- D 项 = −Kd × (velocity − prev_velocity) / dt。

### 4.3 位置速度模式（VELOCITY_POSITION_MODE）

- 通过 REG_TARGET_SPEED（0x0006~0x0007）设置目标速度。
- 固件每 5ms PID 周期执行目标位置累加：
  ```
  target_pos_frac += target_speed * dt;     // dt = 0.005s
  inc = (int64_t)target_pos_frac;
  target_position += inc;
  target_pos_frac -= (float)inc;
  ```
- 走位置环 PID（pos_Kp/Ki/Kd），不是速度环。
- 不走死区：目标持续移动，死区会导致脉冲式运动。

### 4.4 外部目标位置模式（EXTERNAL_TARGET_MODE）

- 通过写 `0x0000 = 4` 切换到外部目标位置模式。
- PB4/PB5 对应输入功能需要配置为 `4=目标位置电平`。
- PB4 有效时，目标位置写入 PB4 预设目标位置（0x002A~0x002D）。
- PB4 无效且 PB5 有效时，目标位置写入 PB5 预设目标位置（0x002E~0x0031）。
- PB4/PB5 都无效时，目标位置写入 0，回到原点位置。
- PB4 优先级高于 PB5。
- 该模式走位置环 PID，并使用位置模式死区。

### 5.3 模式设置寄存器（0x0000）

0x0000 只表示目标工作模式，固件只取低 4 位：

| 写入值 | 模式 |
|---|---|
| 0 | 位置模式 |
| 1 | 速度模式 |
| 2 | 位置速度模式 |
| 3 | 开环模式 |
| 4 | 外部目标位置模式 |

写入 0x0000 时：
- 若 0x0001=0（直接启动），立即切换到对应模式并应用当前目标值。
- 若 0x0001=1（标志位启动），只缓存目标模式，不立即启动；需要再写 0x0010=1 才启动。

### 5.4 启动方式与启动触发

#### 0x0001：是否使用标志位启动

| 写入值 | 含义 |
|---|---|
| 0 | 直接启动。写 0x0000 或目标位置/速度后立即生效。 |
| 1 | 标志位启动。写 0x0000 和目标位置/速度只缓存，写 0x0010=1 后才启动。 |

0x0001 会保存到 Flash。

#### 0x0010：启动触发寄存器

| 写入值 | 行为 |
|---|---|
| 1 | 当 0x0001=1 时，按 0x0000 缓存的模式启动，并应用缓存目标值。 |
| 0 | 清除启动状态，不主动刹停电机。 |

0x0010 不保存到 Flash。直接启动模式（0x0001=0）下，通常不需要写 0x0010。

### 5.5 从机地址寄存器（0x0014）

> 另见 5.6 节的波特率寄存器（0x0015），用于配置 MODBUS 通信波特率。

| 写入值 | 行为 |
|---|---|
| 1~247 | 设置当前 MODBUS 从机地址，立即生效并保存到 Flash |
| 0 或 >247 | 忽略，不修改当前地址 |

读取 0x0014 返回当前正在使用的从机地址。

> **重要：** 写 0x0014 修改地址时，响应帧仍使用修改前的旧地址返回，保证主机能收到写入确认；确认完成后，后续通信必须使用新地址。该地址会保存到 Flash，设备复位后优先使用 Flash 保存的地址；若 Flash 中无有效地址，则使用 [main.c](file:///c:/Users/wangs/Desktop/STM32/sifudianji/rb/Core/Src/main.c#L113) 中 `MODBUS_Init(&mb_ctx, 1, &motor)` 设置的初始化地址 1。

### 5.6 设置原点寄存器（0x0016）

| 写入值 | 行为 |
|---|---|
| 1 | 触发「设置当前位置为原点」：编码器清零，目标位置同步为 0，PID 积分项与速度滤波缓冲清零 |
| 其他 | 忽略，不触发任何动作 |

读取 0x0016 始终返回 0（只写触发寄存器）。

执行后固件内部会做以下同步（在 TIM2 中断关闭的临界区内完成，避免与 PID 运算并发）：

1. `Encoder_Reset()` —— 编码器 64 位计数清零
2. `target_position = 0`、`target_pos_frac = 0` —— 目标位置同步为 0，防止电机突然转动
3. `prev_position = 0` —— 防止下一次速度计算出现巨大尖峰
4. `integral = 0`、`prev_error = 0`、`prev_velocity = 0` —— 清零 PID 积分与速度微分状态
5. 速度移动平均滤波缓冲清零

MODBUS 侧缓存的 `pending_target_pos`、`pending_target_speed` 也会被同步清零，避免下次应用目标值时仍使用旧坐标。

> **注意：** 设为原点不会改变电机方向、编码器方向等其它配置；当前位置会被立即清零，目标位置也同步为 0，因此位置模式下电机会保持在当前物理位置（原点）。该动作不保存到 Flash，复位后位置由系统启动逻辑决定。

## 6. 数据格式与编码

### 6.1 多字节整数

所有多字节整数采用 **大端序** 传输（高字节在前）。

### 6.2 64 位目标位置

由 4 个寄存器拼装，符号为有符号 64 位（int64_t）：

```
position = (REG_H3 << 48) | (REG_H2 << 32) | (REG_L2 << 16) | REG_L1
若 position >= 0x8000000000000000，则 position -= 2^64（转负数）
```

负数写入时主机需先做 `pos += 2^64` 转为无符号再分字节发送。

### 6.3 32 位目标速度

由 2 个寄存器拼装，有符号 32 位（int32_t）：

```
speed = (REG_H << 16) | REG_L
若 speed >= 0x80000000，则 speed -= 2^32
```

### 6.4 32 位最大运行速度

由 0x0028~0x0029 两个寄存器拼装，为无符号限速值（固件有效范围 0~1000000）：

```
max_run_speed = (REG_MAX_RUN_SPEED_H << 16) | REG_MAX_RUN_SPEED_L
```

- `0` 表示不限速。
- `>0` 表示所有运行模式的最大速度不能超过该值。
- 推荐使用功能码 0x10 一次性写 0x0028~0x0029，避免单寄存器分次写入时产生临时中间值。

### 6.5 PID 参数（×100 缩放）

- 寄存器按 **16 位有符号整数（int16_t）** 存储，范围 -32768 ~ +32767。
- 实际浮点值 = 寄存器值 / 100.0，对应实际范围 **-327.68 ~ +327.67**，精度 0.01。
- 写入时 STM32 端：`mc->Kp = (float)(int16_t)value / 100.0f`。
- 读取时 STM32 端：`value = (uint16_t)(int16_t)(mc->Kp * 100)`。
- 主机发送负数时需转为 16 位补码：`value += 0x10000`。

### 6.6 PWM 输出

REG_CURRENT_PWM（0x0108）为 16 位有符号，范围 -1000 ~ +1000，对应 -100.0% ~ +100.0%（百分比 = 寄存器值 / 10.0）。

## 7. 典型工作流

### 7.1 启动方式选择

通过 0x0001 配置是否使用标志位启动：

| 0x0001 值 | 模式 | 行为 |
|---|---|---|
| 0 | 直接启动（默认） | 写 0x0000 切换模式，写目标位置/速度后立即生效 |
| 1 | 标志位启动 | 写 0x0000 和目标位置/速度只缓存；写 0x0010=1 后启动 |

### 7.2 设置目标位置（推荐用 0x10 写 4 个寄存器）

```
请求：01 10 00 02 00 04 08  HH HH  HL HL  LH LH  LL LL  CRCL CRCH
响应：01 10 00 02 00 04  CRCL CRCH
```

整帧写入完成后，STM32 检测到写入了 0x0002~0x0005 范围：
- 若 0x0001=0 或 0x0010 已启动，则立即应用目标值。
- 若 0x0001=1 且 0x0010 未启动，则只缓存目标值，等待写 0x0010=1 后应用。
- 当前模式为 **POSITION_MODE** 时，应用为 `MotorControl_SetTarget(pending_target_pos)`。
- 当前模式为 **SPEED_MODE 或 VELOCITY_POSITION_MODE** 时，应用为 `MotorControl_SetSpeed(pending_target_speed)`。

> **注意：** 多字节目标值请使用 0x10 一次性写入，避免单寄存器逐个写入时中途应用半更新数据。

### 7.3 设置目标速度

```
请求：01 10 00 06 00 02 04  HH HH  LL LL  CRCL CRCH
响应：01 10 00 06 00 02  CRCL CRCH
```

位置速度模式下，目标速度生效后 `ApplyTarget` 调用 `MotorControl_SetSpeed`，固件会在后续 5ms PID 周期中按该速度累加目标位置。若 0x0001=1 且 0x0010 未启动，写入目标速度只缓存，不立即生效。

### 7.4 设置最大运行速度

```
请求：01 10 00 28 00 02 04  HH HH  LL LL  CRCL CRCH
响应：01 10 00 28 00 02  CRCL CRCH
```

- 写 `0`：无限制。
- 写 `>0`：位置模式、速度模式、位置速度模式、开环模式、复位运行都会受该速度限制。
- 该参数会保存到 Flash，掉电后保持。

### 7.5 切换模式并启动

#### 直接启动（0x0001=0）

只需要写 0x0000：

| 目标模式 | 写 0x0000 |
|---|---|
| 位置模式 | 0 |
| 速度模式 | 1 |
| 位置速度模式 | 2 |

#### 标志位启动（0x0001=1）

需要两步：

1. 写 0x0000 = 模式值
2. 写 0x0010 = 1 启动

| 目标模式 | 第一步：写 0x0000 | 第二步：写 0x0010 |
|---|---|---|
| 位置模式 | 0 | 1 |
| 速度模式 | 1 | 1 |
| 位置速度模式 | 2 | 1 |

### 7.6 在线调整 PID

```
请求：01 10 00 08 00 03 06  KP_KP KP_KP  KP_KI KP_KI  KP_KD KP_KD  CRCL CRCH
响应：01 10 00 08 00 03  CRCL CRCH
```

写入完成后自动保存到 Flash。

### 7.7 读取全部状态

推荐读取状态寄存器 0x0100~0x0108 共 9 个寄存器，再单独读取 0x0010 获取启动触发状态。

```
请求状态：01 03 01 00 00 09  CRCL CRCH
请求启动：01 03 00 10 00 01  CRCL CRCH
```

### 7.8 位置速度模式使用流程

1. 写 0x0001 = 1，选择标志位启动（如不需要标志位启动可写 0）
2. 写 0x0000 = 2，选择位置速度模式
3. 写 0x0006~0x0007，设置目标速度
4. 写 0x0010 = 1，启动
5. 电机开始以指定速度持续运转，位置环 PID 跟踪累加的目标位置
6. 停止：写目标速度 0；写 0x0010 = 0 只清除启动状态，不会主动刹停电机

### 7.9 修改从机地址

1. 使用当前地址访问设备，例如当前地址为 1。
2. 写 0x0014 = 新地址，例如写 2。
3. 设备用旧地址 1 返回写入确认。
4. 确认完成后，设备立即切换到新地址 2。
5. 上位机后续通信必须改用新地址 2。

地址有效范围为 1~247。写入 0 或大于 247 的值会被固件忽略。

## 8. 地址范围校验

| 功能码 | 合法地址范围 | 越界返回 |
|---|---|---|
| 0x03 | 0x0000~0x0031 或 0x0100~0x0108，且一次读取不能跨区 | 异常码 0x02 |
| 0x06 | addr ≤ 0x0031 | 异常码 0x02 |
| 0x10 | 起始 + 数量 ≤ 0x0032 | 异常码 0x02 |

注意：0x03 可读状态寄存器；0x06 / 0x10 不可写状态寄存器。

## 9. 错误恢复与统计

### 9.1 接收统计

`MB_Context_t` 提供两个调试计数器：

| 字段 | 含义 |
|---|---|
| `rx_count` | 成功接收的帧数（每完成一帧 +1） |
| `error_count` | CRC 校验失败 / 帧堆积丢弃 / 超时未处理时 +1 |

### 9.2 帧堆积保护

若上一帧尚未被主循环处理（`rx_complete == 1`）又收到新一帧，新帧被丢弃并 `error_count++`。

### 9.3 UART 错误恢复

- `USART1_IRQHandler` 开头清除 ORE/FE/NE 错误标志，避免 HAL 因错误中止 DMA 接收。
- `HAL_UART_ErrorCallback` 中重启 DMA 接收。
- 主循环检测 DMA 状态异常时自动重启接收。

```
                      └───────────────┘  └───────────┘
```


### 10.4 写位置环 PID：Kp=7.0, Ki=1.1, Kd=0.1（×100 后为 700, 110, 10）

```
请求: 01 10 00 08 00 03 06  02 BC 00 6E 00 0A  CRCL CRCH
响应: 01 10 00 08 00 03  CRCL CRCH
```

### 10.5 设置模式并启动

```
直接启动模式:
  01 06 00 01 00 00  CRCL CRCH    写 0x0001 = 0
  01 06 00 00 00 01  CRCL CRCH    写 0x0000 = 1，立即进入速度模式

标志位启动模式:
  01 06 00 01 00 01  CRCL CRCH    写 0x0001 = 1
  01 06 00 00 00 02  CRCL CRCH    写 0x0000 = 2，缓存位置速度模式
  01 06 00 10 00 01  CRCL CRCH    写 0x0010 = 1，启动
```

### 10.6 读取 / 清除启动触发状态

```
读取启动触发状态:
  01 03 00 10 00 01  CRCL CRCH

清除启动触发状态:
  01 06 00 10 00 00  CRCL CRCH
```

### 10.7 修改从机地址

```
读取当前地址:
  01 03 00 14 00 01  CRCL CRCH

把地址从 1 改为 2:
  01 06 00 14 00 02  CRCL CRCH

响应仍使用旧地址 1:
  01 06 00 14 00 02  CRCL CRCH

之后主机必须使用地址 2 通信。
```

### 10.8 位置速度模式完整流程

```
① 启用标志位启动:    01 06 00 01 00 01  CRCL CRCH
② 选择位置速度模式:  01 06 00 00 00 02  CRCL CRCH
③ 设置目标速度 2000: 01 10 00 06 00 02 04  00 00 07 D0  CRCL CRCH
④ 启动:              01 06 00 10 00 01  CRCL CRCH
⑤ 停止（速度设为0）: 01 10 00 06 00 02 04  00 00 00 00  CRCL CRCH
```

### 10.9 设置当前位置为原点

写 0x0016 = 1 触发，编码器立即清零，目标位置同步为 0。

```
请求: 01 06 00 16 00 01  CRCL CRCH
响应: 01 06 00 16 00 01  CRCL CRCH   （原样回显）
```

执行后读取 0x0100~0x0103（当前位置）应返回 0。

### 10.10 异常响应示例（读越界）

## 11. 实现注意事项

1. **寄存器语义已重排：** 0x0000 是模式设置，0x0001 是是否使用标志位启动，0x0010 是启动触发。
2. **旧报文不再推荐：** 旧的 `0x8000 | mode` 控制字写法已经废弃，旧的写 0x0013 启动也不再作为主流程；标志位启动新流程是写 0x0001=1、写 0x0000=mode、写 0x0010=1。
3. **单寄存器写 vs 多寄存器写：** 多字节目标值务必用 0x10 一次性写入，避免中途应用半更新数据。
4. **模式切换副作用：** 切到位置模式/位置速度模式时 `pending_target_pos` 同步为当前位置，切到速度模式时 `pending_target_speed` 清零，避免目标值突变引起电机冲位。
5. **位置速度模式不走死区：** 目标位置持续移动，若启用死区会导致脉冲式运动。
6. **PID 负数处理：** 主机发送负 PID 值前需 `value += 0x10000` 转 16 位补码；STM32 端用 `(int16_t)value` 解码。
7. **参数持久化：** 0x0001、0x0008~0x0012、0x0014、0x0015 的写入会触发 Flash 保存，频繁写入会消耗 Flash 擦写寿命；0x0010 不保存。
8. **MAX_OUTPUT 范围保护：** 写入值 <10 或 >1000 时固件强制改为 900，防止误设为 0 导致电机不转。
9. **帧间超时依赖 TIM1：** TIM1 必须以 100µs 周期稳定运行，否则帧边界判定失效。
10. **修改从机地址后要切换主机地址：** 写 0x0014 后，响应帧用旧地址返回；响应完成后设备改用新地址并保存到 Flash，所以上位机后续读写必须切换到新地址。
11. **设置原点（0x0016）是瞬时动作：** 写 1 触发编码器清零与目标位置同步，固件在 TIM2 中断关闭的临界区内执行，避免与 PID 运算并发；该动作不保存到 Flash，复位后不保留。读取始终返回 0。
